# Voice, images, and memos

Sometimes the useful context is a spoken observation, a screenshot, or an image instead of a typed request. Computer can keep those inputs with the workspace so a review note does not become a disconnected message somewhere else.

## Choose the outcome you need

- Use voice chat when speaking the next agent request is easier than typing it.
- Use a voice memo when you want a recording saved with the workspace. With speech-to-text configured, it can also produce a transcript.
- Use image input or image generation when the selected provider supports it and the result belongs with the project.

## Capture a small test first

1. Configure the required audio, speech-to-text, text-to-speech, or image provider in Settings.
2. Record a short memo or send one harmless image/request.
3. Open the resulting chat item or workspace file and confirm it is associated with the intended project before using it in a broader task.

## Check the result

The recording or image result is visible in the expected chat or workspace location. When transcription is enabled and available, the memo has a transcript; otherwise the audio remains saved as audio. If capture fails, check device permission and the configured provider before recording again.

## What to trust

Hosted speech and image providers can receive media or prompts. A browser speech fallback may have its own browser data policy. Treat recordings, screenshots, and generated images as project data and do not capture secrets or another person's private material without permission.
