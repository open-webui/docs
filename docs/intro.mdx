## What is Open WebUI?

A self-hosted AI platform that puts you in control of the entire stack.

Connect to local models through [Ollama](getting-started\quick-start\starting-with-ollama.mdx), remote providers through [OpenAI-compatible APIs](getting-started\quick-start\starting-with-openai-compatible.mdx), or use [Functions](docs\getting-started\quick-start\starting-with-functions.mdx) to connect directly to Anthropic, Google, and others‚Äîall through a single interface you own. Swap providers on the fly without changing workflows. Your conversations, your documents, your prompts‚Äîthey stay yours.

Open WebUI handles more than chat. Image generation through [ComfyUI](features\image-generation-and-editing\comfyui.md) or [AUTOMATIC1111](features\image-generation-and-editing\automatic1111.md) (or external services like Gemini, DALL-E, and more). Speech-to-text. Voice synthesis. [RAG pipelines](features\rag\index.md) over your documents. One platform, one interface‚Äîwhether you're running everything locally or routing to external services.

Run it as a server and access it securely across all your devices. Add users when you're ready. Set permissions. Keep everything on infrastructure you control.

**You decide what stays local and what reaches out.** Air-gap it completely, connect only to local models, or mix local and cloud providers based on sensitivity. The platform doesn't phone home‚Äîwhat you connect it to is always your call.

## Quick Start with Docker üê≥

Pick your path. All commands are copy-paste ready.

:::info

**WebSocket** support is required for Open WebUI to function correctly. Ensure that your network configuration allows WebSocket connections.

:::

import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';

<Tabs>
  <TabItem value="standalone" label="Open WebUI" default>

**Use this if:** You already have Ollama running locally, or you're connecting exclusively through external APIs ( OpenRouter, OpenAI, Anthropic, Azure, etc.)

<Tabs>
  <TabItem value="gpu" label="With Nvidia GPU" default>

Use this if you have an Nvidia GPU and want GPU-accelerated features in Open WebUI.

```bash
docker run -d -p 3000:8080 --gpus all \
  --add-host=host.docker.internal:host-gateway \
  -v open-webui:/app/backend/data \
  --name open-webui \
  --restart always \
  ghcr.io/open-webui/open-webui:cuda
```

  </TabItem>
  <TabItem value="cpu" label="CPU Only">

No Nvidia GPU? This works on any machine.

```bash
docker run -d -p 3000:8080 \
  --add-host=host.docker.internal:host-gateway \
  -v open-webui:/app/backend/data \
  --name open-webui \
  --restart always \
  ghcr.io/open-webui/open-webui:main
```

  </TabItem>
</Tabs>

  </TabItem>
  <TabItem value="bundled" label="Open WebUI + Ollama">

**Use this if:** You want everything in one container. Open WebUI will install and manage Ollama for you.

<Tabs>
  <TabItem value="gpu" label="With GPU" default>

Use this if you have a GPU. Ollama will use it for faster model inference.

```bash
docker run -d -p 3000:8080 --gpus all \
  -v ollama:/root/.ollama \
  -v open-webui:/app/backend/data \
  --name open-webui \
  --restart always \
  ghcr.io/open-webui/open-webui:ollama
```

  </TabItem>
  <TabItem value="cpu" label="CPU Only">

No GPU? Ollama runs on CPU too‚Äîslower inference, but it works.

```bash
docker run -d -p 3000:8080 \
  -v ollama:/root/.ollama \
  -v open-webui:/app/backend/data \
  --name open-webui \
  --restart always \
  ghcr.io/open-webui/open-webui:ollama
```

  </TabItem>
</Tabs>

  </TabItem>
</Tabs>

Once it's running, open [localhost:3000](http://localhost:3000). That's it‚Äîyou're in.

## What You Can Do

### Connect Any Model

Ollama, OpenAI, Azure, Anthropic, local GGUF files, OpenAI-compatible APIs‚Äîbring whatever models you want. Mix local and remote models for different use cases. Compare models side by side. Switch between them mid-conversation.

[Connect your first model ‚Üí](/getting-started/quick-start)

### Ground Responses with _Your_ Context

Upload documents. Build knowledge bases. RAG is built in‚Äîno external vector database required (though you can bring one if you want).

[Deep dive into RAG ‚Üí](/features/rag)

### Actually Own Your Data

Your conversations are stored locally. Your documents stay on your infrastructure. We don't phone home. We don't collect telemetry. We literally don't know what you're building. You own your data. You own your models. You own your conversations.

### Run Anywhere

Docker, Kubernetes, bare metal, your laptop, an air-gapped server in a basement. GPU acceleration when you have it, CPU-only when you don't.
